У java проекта есть структура, которой надо следовать.

В корне есть папка src, в которой есть разделение на main и test.
Код с бизнес логикой пишем в main, а тесты, которые тестируют этот код - в test.

Например, в main у нас описана функция сложения в классе Main, в test у нас описан юнит тест в классе MainTest.
Можно сделать MainTest автоматически, если в классе Main нажать на название класса альт+Ентер и выбрать
    "создать тест"

Такое разделение нужно для того, чтобы:
1) Чтобы код тестов и зависимоестей для тестов не попадали в итоговую сборку проекта.
2) Зависит от скопов, если в гредле в dependencies написано implementations, то зависимость будет работать в
    обоих папках. А вот  testimplementations будет работать только в папке test

Созданный класс должен называться как файл и с большой буквы.

public static void main(String[] args) {} - это сигнатура, совокупность характеристик метода, состоящих из
    модификатора доступа + static + из того, что он возвращает (void - ничего не возвращает) + имя метода
    + параметра

Слово class в Джаве - это определение нового типа данных. Поэтому ответ на вопрос "Сколько типов данных"
    в Джаве будет бесконечное количество (8 примитивных + можно создавать свои при помощи class
    (объектные или ссылочные))

Идея классов - добавить какое-то повидение, создав в классе метод.

Тестовый сценарий подразумевает отсутствие ветвления логики (никаких if else)
